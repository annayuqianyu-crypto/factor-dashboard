# 痛点向量大更新任务包

## 任务唤醒词

启动痛点向量大更新

## 唤醒后要执行的完整任务

当用户输入上述唤醒词时，默认理解为：

基于最新人工审核完成的痛点主库，重新完成痛点向量化、法税资三维回算、族簇重算、MECE 与向量断裂审计、补洞场景生成、驾驶舱页面更新、本地同步和 GitHub Pages 部署。

## 输入文件

1. 最新人工审核完成的痛点库 Excel。
2. 已同步痛点字段，包括但不限于：
   - 痛点编号
   - 归属 BU
   - 法律法规
   - 痛点描述
   - Tags：主体、资产、动作、约束、风险
   - 核心冲突
   - 潜在风险
   - 解决方案
   - 预期效果
   - 匹配 SKU
3. 当前驾驶舱页面：`index.html` / `dashboard.html`。
4. 本地 BGE-M3 向量模型与已定义专家锚点。

## 涉及 skill / 能力清单

### 1. 表格读取与数据质检

用途：

- 读取最新痛点主库。
- 校验字段缺失、编号异常、重复行、空标签、异常 BU。
- 保证所有真实痛点编号仍为 `P-xxx` 体系。

输出：

- 痛点主库质检摘要。
- 编号异常表。
- 字段缺失表。

### 2. BGE-M3 痛点向量化

用途：

- 将每条痛点合成为统一语义文本。
- 使用 BGE-M3 生成 1024 维向量。
- 输出 `.npy` 矩阵和向量版 Excel。

输出：

- 痛点向量矩阵。
- 痛点向量版 Excel。
- 每条痛点的高维语义向量。

### 3. 专家锚点与 L/T/C 三维回算

用途：

- 使用法律、税务、资本市场三条专家锚点作为三维坐标尺。
- 计算每条痛点与三条锚点的余弦相似度。
- 将相似度拉伸为可读的 `L_score / T_score / C_score`。

输出：

- L/T/C 三轴强度。
- 主投影面：L、T、C、LT、LC、TC、LTC。
- 法税资三维坐标表。

### 4. 痛点簇与痛点族重算

用途：

- 先形成痛点簇：局部高密度小问题。
- 再形成痛点族：多个同源簇向上归并后的一级业务问题域。
- 结合母痛点编号、SKU、核心冲突、向量相似度和 L/T/C 结构共同判断。

输出：

- 痛点簇表。
- 痛点族表。
- 族-簇-痛点映射表。

### 5. MECE 与向量审计

用途：

- 查重复：识别向量相似度过高、可能重复的痛点。
- 查断裂：识别同族 / 同簇内部语义不连续的位置。
- 查缺口：识别样本稀疏、适合补洞的向量区域。

输出：

- 疑似重复痛点审计报告。
- 向量断裂审计报告。
- 业务化补全建议。

### 6. 补洞场景生成

用途：

- 从向量缺口反推候选业务场景。
- 每条场景要带目标族、目标簇、L/T/C 位置、审核归口 BU、协同 BU 和后台补洞约束。

输出：

- 补洞场景长表。
- 可推送到“场景触发”页的场景输入文本。

### 7. 驾驶舱页面更新

用途：

- 更新管理看板中的痛点总数、周度变化、关键事件。
- 更新痛点向量分析页面中的族、簇、代表痛点、向量缺口和补洞场景。
- 保持页面风格、目录逻辑和 BU 使用习惯不大改。

输出：

- 更新后的 `index.html`。
- 同步后的本地 `dashboard.html`。

### 8. 部署与归档

用途：

- 将页面同步到本地驾驶舱。
- 提交并推送 GitHub Pages。
- 将本轮 Excel、npy、审计报告、补洞场景长表放入对应月份文件夹。

输出：

- GitHub Pages 最新版本。
- 本地归档文件夹。
- 本轮更新提交记录。

## 执行原则

1. 人工修改完成的痛点库是唯一可信基准。
2. 不沿用旧版 737 条痛点结论，必须基于最新主库重新计算。
3. 不伪造 `P-xxx` 痛点编号；模型族 / 簇编号不得冒充痛点编号。
4. 数学结论必须翻译成业务建议，不能只输出抽象相似度或距离。
5. 补洞场景必须可直接推送到“场景触发”页生成痛点。
6. 更新驾驶舱时只做必要轻改，不重构 BU 已习惯的页面逻辑。

## 推荐归档位置

每次大更新建议新建：

`08 痛点数据 / YYYYMM痛点向量化分析`

其中包含：

- 痛点向量版 Excel
- `.npy` 向量矩阵
- 族簇映射表
- 重复性审计报告
- 向量断裂业务审计报告
- 补洞场景长表
- 驾驶舱更新说明
